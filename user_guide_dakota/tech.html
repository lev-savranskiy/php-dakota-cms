Dakota CMS, руководство пользователя к версии 0.4

Dakota CMS - основана на лидирующих технологиях

CodeIgniter

CodeIgniter очень легкий. Основное ядро требует небольшого количества библиотек. Это отличает Dakota CMS от большинства CMS, которые требуют значительно больше ресурсов. Дополнительные библиотеки загружаются динамически по запросу, в зависимости от потребностей в тот или иной момент, так что ядро остается очень гибким и быстрым.

ORM Doctrine

ORM избавляет программиста от написания большого количества кода, часто однообразного и подверженного ошибкам, тем самым значительно повышается скорость разработки. Одна из ведущих современных ORM - Doctrine позволяет работать с данными в базе как с реально существующими обьектами и загружать данные из YAML файлов, что и реализовано при установке Dakota CMS.

Гибридная шаблонизация

Каждая страница уже сверстана и разбита на общепринятые стандартные части footer, header, content, sidebar. Для Dakota CMS подходят тысячи бесплатных шаблонов WordPress с минимальными правками. С помощью шаблонов, javascript и flash вы можете полностью изменить внешний вид проекта.

Перенос логики на сторону фронтенда

На каждой странице подключен популярный фреймворк Jquery. Все данные о текущем юзере доступны в javascript Object userdata. И это только первый шаг. В будущем планируется развитие такого подхода - перенос части логики на сторону фронтенда без сложных шаблонизаторов. В итоге планируется дать создателю сайта выбор - получать традиционные HTML данные от CMS ядра и преобразовывать их шаблонами, или получать все данные в javascript и рисовать их полностью JavaScript без шаблонизаторов в принятом смысле.

Безопасность

Безопасность - это главная составляющая любого веб-проекта. Dakota CMS поставляется с фильтром, предотвращающим XSS атаки (кроме Админ-панели) и CSRF атаки (везде).
