Dakota CMS, руководство пользователя к версии 0.4

Основные настройки

Данные сайта

  • Здесь Вы можете изменить TITLE, KEYWORDS, META_DESCRIPTION сайта. Писать нужно текст, без тегов.
  • META_EXTRA: дополнительные теги , которые вы хотите вывести в HEAD. Писать нужно HTML разметку, например

    ‹meta name="google-site-verification" content="asfdsf3w-XWDNFjTNJnFS6w66CrL1pb5a8_2vbe05w" /›

    Оставьте пустым, если не знаете , что это.
  • Статус #profile-link на сайте (Если Вам не нужен раздел авторизации #profile-link или вы хотите кастомизировать его с помощью javascript).
  • Статус регистрации на сайте.
    • Возможность входа по ссылке login сохраняется всегда.

Работа с кэшем

  • Указав время кеширования, вы можете существенно ускорить загрузку этих страниц сайта.
    • Публикации с отключенными комментариями
    • Список пользователей
    • Галерея
    • Если кэш включен, то все изменения в указанных разделах не будут видны до его истечения. Для удаления кэша воспользуйтесь одноименной ссылкой.

Работа с БД

  • Кнопка 'Сохранить БД в SQL' экспортирует данные SQL в файл mybackup.sql в корне сайта и на рабочий стол с названием, содержащим текущее время. Это может быть удобно для миграции и архивирования сайта.
  • В текущей версии Дакота сохраняет только последнюю версию БД. Если вам нужно ежедневное копирование, обратитесь к администратору сервера.
  • Кнопка 'Перестроить БД' запросит ссылку /admin/load_fixtures/. Таким образом Все таблицы будут удалены и созданы из моделей Doctrine_Record, папка /application/models. Все данные будут удалены и загружены из папки /application/fixtures
    • Это действие нельзя отменить из админ-центра. Рекомендуется Сохранить БД в SQL файл и иметь познания в SQL. Удалите файл /admin/load_fixtures.php, если вы не планируете обнулить систему или заниматься собственной разработкой c YAML фикстурами.
  • Если при загрузке /admin/load_fixtures/ произошла ошибка, удалите из БД вручную все таблицы, кроме "ci_sessions" и попробуйте еще раз.
  • Все операции с БД не влияют на виджеты, шаблоны, галерею и файлы в папке uploads

Работа с Sitemap

  • Sitemaps — это XML-файл с информацией для поисковых систем о страницах веб-сайта, которые подлежат индексации.
  • Кнопка 'Перестроить sitemap' создаёт новый sitemap.xml в корне сайта с информацией о текущих публикациях а также отсылает sitemap на Google, Ask, bing, Yahoo
  • В текущей версии Dakota создает sitemap из основных разделов с приоритетом 0,5, обновление daily и всех видимых публикаций с приоритетом 0.4, обновление monthly
  • Для добавления sitemap в Яндекс перейдите по ссылке http://webmaster.yandex.ru/site/add.xml
